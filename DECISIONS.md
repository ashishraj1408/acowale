# Architecture Decisions

## Backend
- Used Express with a modular MVC-style structure for clear separation of concerns.
- Implemented Prisma for database access and schema management.
- Centralized validation and error handling for consistent API responses.

## Frontend
- Used React Router for multi-page navigation.
- Used Tailwind CSS for a professional responsive UI.
- Used React Hot Toast for friendly feedback notifications.

## Validation
- Enforced category and comment rules on both the client and backend.
- Used Zod to keep validation definitions consistent.
